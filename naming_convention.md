# Naming convention

- Acronyms are considered as single words.
  - e.g. `Cpu`, `Ast`, etc.

## Variables and types

- Lower or upper camel case
  - `fooBar`
  - `FooBar`

## Modules

- Snake case
  - `foo_bar`

## Module directories

- Lower case without signs
  - `foo_bar`
- Follow [Go's convention](https://blog.golang.org/package-names).

## Packages

- Kebab case
  - `github.com/foo-bar/baz-blah`
